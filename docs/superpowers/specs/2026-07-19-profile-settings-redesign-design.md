# 个人设置页 GitHub 式重构设计

## 背景与目标

当前 `/workspace/profile` 将头像、公开资料和密码修改放在纵向卡片中，内容宽度较窄，信息层级与工作区其他页面脱节。本次重构在不改变现有后端接口和数据字段的前提下，引入 GitHub Settings 的账户摘要、左侧设置导航和右侧分区表单结构，同时保留 AutoforgeHub 的橙色品牌、深浅主题和现有字体体系。

成功标准：

- 公开资料与密码安全位于同一设置页，并可通过左侧导航快速定位。
- 桌面端呈现清晰的设置双栏，移动端无横向溢出且保持完整操作能力。
- 头像上传裁剪、名称保存、只读邮箱和密码修改行为保持可用。
- 页面沿用现有主题变量，视觉更克制、紧凑，接近 GitHub Settings 的信息密度。
- 表单反馈对视觉用户和辅助技术均清晰可辨。

## 信息架构

页面继续使用 `WorkspaceWsHeader`，主体宽度约为 `1100px`，按以下层级组织：

1. 设置主体：桌面端为约 `240px` 的左侧导航和自适应右侧内容区。
2. 左侧导航：包含“公开资料”和“密码与安全”两个锚点项。
3. 公开资料区：左侧为显示名称、只读邮箱和保存操作，右侧为大头像与更换入口。
4. 密码与安全区：旧密码、新密码、确认密码、密码要求提示与更新操作。

左侧导航在桌面端吸顶；窄屏下转换为主体上方的横向分段导航。内容区不使用厚重的整块卡片，而以标题、分隔线、紧凑字段和局部反馈建立层级。

## 视觉方向

采用“GitHub 式结构，AutoforgeHub 品牌表达”：

- 保留现有 `--accent` 橙色、深浅主题变量和 `--font-display` 品牌字体。
- 减少大圆角、强阴影和大面积渐变，边框与分隔线成为主要层级工具。
- 主按钮使用紧凑内容宽度，保留橙色强调，但取消全宽布局。
- 输入框使用较小圆角、清晰边框和一致的焦点环。
- 头像更换入口采用覆盖式“编辑”按钮，保持裁剪流程不变。
- 动效只用于导航高亮、按钮状态和区块进入视口等微交互，并尊重 `prefers-reduced-motion`。

## 组件与职责

本次优先在 `app/pages/workspace/profile.vue` 内完成聚焦式重构，避免为两个导航项引入不必要的组件层级。页面内部职责如下：

- 账户摘要：读取 `useAuth()` 中的用户和头像状态。
- 设置导航：维护当前区块键值，处理锚点点击与滚动后的激活状态。
- 公开资料表单：校验显示名称，调用 `PUT /api/auth/profile`，成功后调用 `updateUser()` 同步全局用户状态。
- 头像编辑：继续使用现有文件选择、`WorkspaceAvatarCropModal` 和 `POST /api/auth/avatar` 流程。
- 密码表单：继续调用 `changePassword()`，保留长度和一致性校验。

若实施时页面文件因模板和样式明显失控，仅允许将左侧导航抽为 `app/components/workspace/ProfileSettingsNav.vue`；业务表单和请求逻辑仍保留在页面中。

## 数据流与状态

页面挂载时通过 `/api/auth/me` 同步显示名称和头像。公开资料保存成功后，使用接口返回的 `displayName` 更新页面字段与全局用户状态，使顶部账户菜单立即反映新名称。头像上传成功后继续使用返回的 `avatarUrl` 更新本地与全局状态。

公开资料和密码表单分别维护提交中、成功和失败状态，互不阻塞。导航激活状态只影响表现，不参与业务数据提交。所有新增状态均为页面本地状态，不新增持久化字段。

## 交互与可访问性

- 导航项使用可聚焦按钮或链接，并通过 `aria-current` 表示当前区块。
- 锚点滚动考虑吸顶工作区头部的高度，避免标题被遮挡。
- 表单标签与输入框显式关联；只读邮箱保持不可编辑并说明原因。
- 成功信息使用 `role="status"`，错误信息使用 `role="alert"`。
- 提交期间禁用对应按钮和字段，并保留清晰的加载文案与图标。
- 头像编辑按钮提供明确文本或无障碍名称，不依赖单独的相机图标传达含义。
- 移动端顺序为账户摘要、导航、头像、公开资料、密码安全，保持自然阅读流。

## 错误处理

- 空显示名称在请求前拦截，保留服务端错误消息作为最终依据。
- 资料请求成功后才更新全局用户状态；失败时不改变当前账户信息。
- 密码为空、长度不足和两次输入不一致均在本地提示；服务端错误原样展示。
- 头像上传与资料保存共享公开资料区反馈，但每次新操作会清理旧消息，避免误导。
- `/api/auth/me` 初始化失败保持静默，继续使用现有会话中的用户数据。

## 响应式设计

- `>= 900px`：双栏设置布局，左侧导航吸顶；公开资料内部为表单与头像双栏。
- `< 900px`：导航移至内容顶部，公开资料缩为单栏，头像置于表单前。
- `< 640px`：收紧页面边距与字段间距，按钮在必要时可扩展为整行，确保触控区域充足。

## 验证方案

实施后进行以下验证：

- ESLint 检查目标 Vue 文件及相关抽取组件。
- Nuxt 生产构建，验证模板、类型和样式编译。
- 桌面、平板和手机宽度下检查导航、头像、表单与反馈布局。
- 深色与浅色主题下检查文本、边框、焦点和按钮对比度。
- 手动验证名称保存后顶栏同步、头像裁剪上传、密码校验与成功/失败反馈。

## 非目标

- 不新增个人简介、社交链接、公司、地点等后端尚未支持的字段。
- 不拆分新的设置路由，不修改数据库结构或认证接口。
- 不重构 `WorkspaceWsHeader`、头像裁剪弹窗或全局设计令牌。
- 不照搬 GitHub 的颜色、品牌标识或完整设置菜单。
