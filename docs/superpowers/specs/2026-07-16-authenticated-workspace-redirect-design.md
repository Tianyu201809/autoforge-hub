# 有效登录态直接进入工作区设计

## 目标

当客户端确认当前登录状态有效时，访问登录页应直接跳转到 `/workspace`，不再经过 `/` 中转，也不读取登录页的 `redirect` 查询参数。

## 路由行为

- 已登录用户访问 `/login` 时，使用替换历史记录的方式跳转到 `/workspace`。
- 已登录用户访问 `/login?redirect=...` 时，仍固定跳转到 `/workspace`。
- 未登录、登录信息无效或已过期的用户继续停留在 `/login`。
- 正常登录成功后的现有跳转、受保护路由的访客拦截以及 `401` 清理登录状态逻辑保持不变。

## 实现范围

- 仅修改全局认证路由中间件中“公开路由且已登录”的分支，将目标从 `/` 改为 `/workspace`，并使用 `replace`。
- 不在登录页或认证插件中新增重复监听，不修改登录 API、会话存储、Cookie 或页面布局。

## 验证

- 有效登录态访问 `/login`，确认直接进入 `/workspace` 且后退不会返回登录页。
- 有效登录态访问带 `redirect` 参数的登录页，确认仍进入 `/workspace`。
- 无有效登录态访问 `/login`，确认登录页正常显示。
- 运行生产构建，确认中间件与路由代码编译通过。
