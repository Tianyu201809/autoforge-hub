# 工作空间脚本分布饼图设计

## 目标

将个人空间和团队空间侧栏中的“脚本分布”列表升级为单个可切换维度的环形饼图，以更直观地呈现脚本分类与语言占比，同时保留原有筛选能力。

## 范围

- 个人空间与团队空间共用同一套饼图组件和交互。
- 支持“分类”和“语言”两个统计维度，通过分段按钮切换。
- 分布统计覆盖当前空间内符合搜索条件及另一维度筛选条件的全部脚本，不受列表分页或懒加载数量影响。
- 不引入第三方图表库，不调整脚本卡片、上传流程、权限体系或其他侧栏模块。

## 数据设计

脚本列表接口在现有分页结果之外返回 `distributions`：

- `category`：按分类名称聚合的脚本数量。
- `language`：按语言名称聚合的脚本数量。

两个维度采用分面统计规则：

- 分类统计应用空间范围、团队、搜索词和语言筛选，但忽略当前分类筛选。
- 语言统计应用空间范围、团队、搜索词和分类筛选，但忽略当前语言筛选。

因此，点击某个扇区后，该维度的完整分布仍然可见；另一维度会根据已选条件实时更新。个人空间只统计当前用户脚本，团队空间只统计当前团队脚本，并沿用现有访问权限校验。

`useScripts` 保存接口返回的分布数据，个人空间和团队空间页面将其转换为现有 `filterGroups` 数据结构。列表请求失败时沿用现有错误处理，并清空分布数据，避免展示过期统计。

## 组件与视觉

`WsSpaceInsightSidebar.vue` 继续负责“脚本分布”模块，并在内部完成 SVG 环形图所需的扇区角度、路径、占比和颜色计算。

- 模块顶部使用“分类 / 语言”分段按钮切换当前维度。
- 图表使用原生 SVG 环形结构，中心显示当前维度的脚本总数和“脚本”标签。
- 扇区采用项目主题色衍生的稳定色板，同一条目在个人与团队空间保持一致颜色。
- 图表下方显示图例，每项包含颜色、名称、数量和百分比。
- 当前筛选项保持高亮；存在筛选时，其他扇区和图例降低视觉权重。
- 个人与团队侧栏沿用现有背景、边框、圆角和响应式断点，不改变整体页面宽度。

## 交互与无障碍

- 点击扇区或图例触发原有 `filter` 事件，筛选对应脚本。
- 再次点击当前项取消筛选。
- 鼠标悬浮或键盘聚焦时突出扇区，并展示名称、数量和占比提示。
- 分段按钮、扇区和图例均提供明确的可访问名称、选中状态和键盘焦点样式。
- SVG 提供图表标题与说明；颜色不是唯一信息载体，图例文字同步表达数据。
- 尊重 `prefers-reduced-motion`，减少或关闭扇区过渡动画。

## 边界状态

- 当前维度没有数据时，不渲染空图，显示现有风格的上传提示。
- 单一数据项占比为 100% 时仍显示完整圆环和对应图例。
- 数量为零的预定义分类或语言不进入饼图；当前选中项即使数量变为零，也保留可取消的筛选入口。
- 条目较多时图例纵向排列并允许自然扩展，不截断名称和数量。
