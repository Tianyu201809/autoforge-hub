# 工作空间快捷操作与权限 Icon Dock 设计

## 目标

将个人空间、团队空间侧栏中的快捷操作，以及团队空间的“我的权限”区域，重构为纯图标展示。新设计减少文字按钮造成的视觉重量，在窄侧栏内建立更紧凑、清晰的操作入口，同时完整保留现有行为与可访问语义。

## 范围

- 个人空间快捷操作：上传脚本、查看上传规范、前往脚本集市、切换团队空间、下载 Autoforge。
- 团队空间快捷操作：上传团队脚本、切换个人空间。
- 团队空间权限状态：上传、编辑、下载、删除。
- 只调整共享组件 `WsSpaceInsightSidebar.vue` 的快捷操作与权限模板、交互提示和 scoped CSS。
- 不改变页面传入的数据结构、点击事件、路由、外链、权限判断、脚本分布饼图及其他侧栏模块。

## 快捷操作 Icon Dock

- 移除现有“快捷操作”文字标题栏和纵向文字按钮列表，整个面板只显示图标入口。
- 个人空间使用三列网格容纳五个入口；团队空间根据两个入口自适应为两列。
- 每个入口使用 42–46px 圆角方形底座，图标在底座中居中，保证触控与键盘焦点区域不小于 42px。
- 主操作使用当前空间强调色与柔和光晕；个人空间使用橙色，团队空间使用紫色。
- 普通操作使用低对比背景、细边框和轻微悬浮位移，不显示常驻文字。
- 外链入口在右上角显示迷你外链箭头角标。
- 禁用入口保持不可操作，降低透明度并在右上角显示锁角标。

## 团队权限图标阵列

- 移除“我的权限”文字标题栏、权限名称和“可用/关闭”胶囊标签。
- 四项权限使用四列等宽图标阵列，分别保留上传、编辑、下载、删除的原有图标。
- 可用权限使用团队紫色描边、柔和背景和右下角对勾状态点。
- 不可用权限使用弱化灰色、虚线或低对比边框，并在右下角显示锁状态点。
- 权限图标仅表达状态，不可点击，不改变现有权限逻辑。

## 提示与无障碍

- 操作名称与权限名称默认完全隐藏。
- 鼠标悬浮或键盘聚焦时显示自定义气泡提示；快捷操作显示名称，权限显示“名称 · 可用”或“名称 · 关闭”。
- 每个操作保留明确的 `aria-label`；权限项使用包含名称和状态的 `aria-label`，并设置 `tabindex="0"` 以支持键盘聚焦查看状态提示。
- 所有可交互入口提供可见的 `:focus-visible` 描边。
- 同时设置原生 `title`，为不支持自定义悬浮气泡的环境提供降级提示。
- 状态不能只依赖颜色：外链、禁用、可用和关闭均使用独立角标图形辅助表达。
- 尊重 `prefers-reduced-motion`，关闭图标位移、光晕和气泡过渡。

## 响应式与主题

- 图标网格在现有 260px 侧栏内完整显示，不改变页面侧栏宽度和断点。
- 侧栏在 1100px 以下回到普通文档流后，图标网格仍保持居中和固定触控尺寸。
- 深色与浅色主题全部使用现有 CSS 变量，不新增全局主题令牌。
- 图标气泡应避免溢出侧栏：左侧入口向左对齐、中间入口居中、右侧入口向右对齐。

## 验证

- 在个人空间验证五个操作的图标排列、主操作、外链角标、跳转和气泡提示。
- 在团队空间验证两个快捷操作、上传禁用状态，以及四项权限的可用/关闭视觉状态。
- 验证鼠标悬浮、键盘 Tab、Enter/Space、焦点提示、`aria-label` 和减少动画偏好。
- 验证深色、浅色和窄屏布局，不影响脚本分布饼图与其他侧栏内容。
- 执行组件定向 Lint 和开发构建；全量 Lint 与 TypeScript 检查继续记录仓库既有基线问题，不修改无关文件。

## 后续修订

- 快捷操作 Icon Dock 仅在个人空间显示；团队空间不渲染快捷操作模块。
- 自定义 Tooltip 作为唯一提示来源，不再同时设置原生 `title`；提示向上展开，并提升当前图标的堆叠层级，避免被相邻图标遮挡。
- 团队权限继续使用纯图标阵列，但恢复带盾牌图标的“我的权限”标题栏，与脚本分布模块保持一致。
- 个人空间快捷操作 Icon Dock 恢复带闪电图标的“快捷操作”标题栏；团队空间仍不渲染该模块。
