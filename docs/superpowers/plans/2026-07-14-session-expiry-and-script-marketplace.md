# 登录失效提示与脚本集市实施计划

> 依据 `2026-07-14-session-expiry-and-script-marketplace-design.md`。

1. 新增客户端认证失效状态 composable，提供“显示失效对话框”和“确认后清理会话并登录”的单例操作；状态应防止并发 401 重复打开。
2. 新增全局登录失效对话框组件，并在 `app/app.vue` 挂载；采用项目现有 modal token、不可关闭、按钮点击后才导航。
3. 新增客户端全局 fetch 拦截器：仅监听 `/api/` 的 401 响应并激活认证失效状态，因此现有 composable 与页面的受保护请求获得一致行为，且保持各自的响应解析与错误展示。
4. 让 `useAuth` 继续负责登录态读写，但改为使用共享请求封装并暴露受控的会话清理能力；登录、注册和密码恢复请求显式跳过失效弹窗。
5. 将 `useMarketplace`、`useScripts`、`useTeams`、`useAuditLogs` 的请求接入共享封装，并处理 FormData / 无 JSON 内容等现有请求差异。
6. 将集市详情、提交、个人空间、团队和资料页里直接发出的受保护请求接入共享封装，保证 token 失效处理不依赖具体页面。
7. 全量替换 `app/` 内可见“插件集市”文本和文档标题为“脚本集市”，保留 URL 与接口名称不变。
8. 验证：用 401 响应检查弹窗只出现一次、点击后才登录；确认集市所有入口文案；运行 `npm run build:dev`。
